# 桌面互动软件 - 10_记忆与数据生命周期文档

## 1. 模块概述
本模块是“赛博好友”的大脑海马体与本地档案库。它负责管理所有视觉输入、语音交互、聊天记录以及性格设定的存储与销毁。本模块的设计核心理念为：**“极致的隐私保护、克制的存储开销、以及多重羁绊的存档隔离”**。绝不让软件成为无底洞般的“硬盘刺客”，也绝不保留任何不该看的敏感信息。

## 2. 记忆的分层架构与生命周期 (The 3-Tier Memory System)
系统将数据严格划分为三个生命周期层级，分别对应不同的物理存储策略：

### 2.1 瞬时工作记忆 (Ephemeral Memory / RAM)
* **存储内容**：02 模块抓取的原始屏幕截图（Base64 编码）、05 模块录制的临时语音音频流。
* **生命周期（阅后即焚）**：**绝对禁止落盘（不写入硬盘）**。数据仅存在于内存中，一旦 API 网络请求结束并返回结果，或状态机（08 模块）切回 `IDLE` 状态，该部分内存将被立即强制释放。从物理层面上彻底杜绝“后台偷偷存图/存音”的隐私隐患。

### 2.2 短期情景记忆 (Short-Term Logs)
* **存储内容**：结构化的文字聊天记录（包括用户的语音转写文本、大模型的回复文本、以及附带的动作/情绪标签）。
* **生命周期（双重限制阈值）**：采用本地轻量级数据库（如 SQLite）存储。设立 **“7 天”** 与 **“1000 条”** 的双重最高生命线。当记录存在时间超过 7 天，或总条数突破 1000 条时，系统触发 FIFO（先进先出）清理机制，自动且静默地抹除最旧的记录。
* **API 携带策略**：为控制 Token 成本并保证极速响应，每次向大模型发起新对话请求时，**固定仅携带最近的 5 轮有效对话**作为上下文语境。

### 2.3 长期灵魂记忆 (Long-Term Profile)
* **存储内容**：这是角色的“灵魂核心”。由 04 模块的“50轮总结机制”在后台默默提炼出的精华档案。例如：用户的游戏偏好（喜欢玩《英雄联盟》、《命运2》等）、核心羁绊（确认用户是“哥哥”）、以及雷区或习惯。
* **生命周期**：纯文本格式本地持久化保存（如 `persona_profile.json`），占用空间极小（通常小于 50KB），**永久保存**，直到用户主动执行存档删除操作。

## 3. 多伴侣存档系统 (Multi-Profile Save Slots)
为满足用户在不同游戏或工作场景下的陪伴需求，系统引入“多存档机制”，彻底告别单一人格：
* **数据物理隔离**：支持在前端 UI 创建多个独立的存档槽位（Save Slots）。每个存档拥有完全独立的“短期情景记忆”、“长期灵魂记忆”以及 09 模块中由 AI 生成的专属 Q 版形象档案。
* **存档交互操作**：
  * **新建 (New Game)**：重新唤起引导流程，让 AI 捏造一个全新的形象与初始性格。
  * **无缝切换 (Load)**：在前端设置面板一键切换不同的“赛博好友”，对应读取不同的记忆数据库。
  * **彻底销毁 (Delete)**：一键执行物理级的数据粉碎，永久抹除该伴侣的所有记忆与形象数据，不可逆转。

## 4. 绝对红线：写入前隐私清洗 (Privacy Scrubbing Gateway)
在数据由“瞬时工作记忆”向“短期/长期记忆”转移的过程中，必须经过严格的脱敏拦截：
* **清洗网关**：所有准备写入本地数据库的文本数据，**必须**先经过 03 模块定义的“上下文清洗规则”过滤。
* **特征打码**：如果在读屏转写的文本或用户语音识别文本中，匹配到手机号、邮箱、密码格式、真实姓名等高危隐私特征，系统会在内存层面直接将其替换为占位符 `[敏感信息已隐藏]`。
* **安全落盘**：本地硬盘的数据库中，永远只会存入清洗后的安全纯文本。即使本地文件被意外读取，也不会造成用户的真实隐私泄露。
