# 桌面互动软件 - 07_API 设计文档 (VLM/LLM 接口通信)

## 1. 模块概述
本模块是软件的“网络枢纽”与“快递站”。它负责将本地收集到的所有上下文信息（视觉截图、系统状态、用户提示词）打包，按照大模型厂商（如 Google Gemini、OpenAI）或本地推理服务（如 Ollama）的标准格式发送请求。本模块旨在提供一个稳定、安全且具备自我纠错能力的通信链路，确保挂件的“大脑”始终在线。

## 2. 核心通信机制：整包同步返回 (Blocking & Full-Payload)
为保证业务逻辑的稳定性和开发的敏捷性，本模块默认采用**整包返回（非流式）**的网络请求模式：
* **工作流**：在接收到上游（模块 03、04）组装好的数据包后，本模块发起异步 HTTP 请求。系统将挂起当前对话任务，等待大模型将一整段完整的回复（包括文本和动作情绪标签）全部生成完毕并回传。
* **状态传递**：在等待网络请求的 1~3 秒期间，本模块会向 **06_交互表现模块** 持续发送 `STATE_PROCESSING` 信号，使得挂件在桌面上表现出“托腮思考”或显示加载动画，避免用户误以为软件卡死。

## 3. 密钥管理与热重载 (Key Management & Hot Reloading)
为保障用户资产安全并提供极致的易用性，API 密钥（API Key）的管理遵循以下严格规范：
1. **零硬编码**：绝对禁止在代码内部硬编码任何 API 密钥。密钥统一存储于本地的 `.env` 文件或 `config/settings.json` 中。
2. **内存热重载**：本模块在每次发起请求前，都会动态校验本地配置文件的 Hash 值或最后修改时间。如果用户通过前端 UI 更改了 API Key，本模块将**即刻应用新密钥**，无需重启软件即可使新配置生效，旧密钥的计费瞬间中止。

## 4. 多端路由与容错兜底机制 (Routing & Fallback)
网络环境和大模型服务具有不确定性，本模块内置了高可用性的路由网关与错误拦截器：

### 4.1 主备模型轮询 (Primary-Secondary Routing)
支持在配置文件中预设多个模型端点（Endpoint）。
* **主力节点**：首选高智商的云端视觉大模型（如 Gemini Pro）。
* **备用节点（本地兜底）**：如果用户配置了本地模型服务（如本地运行的 Ollama），当主力节点发生网络超时（Timeout > 5s）或触发限流时，本模块将在 1 秒内自动将请求重定向至本地模型，确保陪伴不中断。

### 4.2 优雅的鉴权与错误拦截 (Graceful Error Handling)
当大模型接口返回 HTTP 错误码（如 `401 Unauthorized` 密钥失效、`429 Too Many Requests` 额度耗尽）时：
* **禁止闪退**：本模块拦截所有底层异常，绝对不允许报错导致软件崩溃。
* **拟人化反馈**：将死板的错误码转化为友好的结构化数据发给 06 模块。例如，遇到 401 错误，挂件会弹窗并配音：“*哎呀，我的 API 密钥好像失效了，能去设置里帮我检查一下吗？*”

## 5. 数据包封装结构 (Payload Assembly Protocol)
本模块直接调用 02 模块的压缩图像和 03、04 模块的文本，不进行二次压缩。统一转换为如下的 JSON 格式发出，并解析大模型返回的结果：

**【向大模型发出的请求示例 (Request)】**
```json
{
  "model": "gemini-3.1-pro",
  "messages": [
    {
      "role": "system",
      "content": "你是用户的妹妹，性格温柔...（来自模块04）"
    },
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "当前窗口是VS Code，请重点看红线报错...（来自模块03）"},
        {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,/9j/4AAQ..."}}
      ]
    }
  ]
}
