# 桌面互动软件 - 03_视觉上下文与元数据定义文档

## 1. 模块概述
本模块扮演“数据封装器”与“上下文解释器”的角色。纯粹的像素图像缺乏逻辑意义，本模块负责将 02 模块（实时屏幕读取）输出的图像，与操作系统的元数据（时间、进程、触发原因等）进行绑定，并根据当前运行的软件环境动态注入“注意力引导”。最终封装成标准化的**视觉上下文负载 (Vision Context Payload)**，为下游的大模型提供充足的情景感知能力与明确的分析主次。

## 2. 焦点优先级与场景配置 (Focus Prioritization & Scenario Profiles)
为了避免大模型“雨露均沾”地描述画面，本模块引入外部配置文件 `scenario_rules.json`。系统会根据当前激活的 `process_name` 或 `window_title`，自动匹配对应的场景规则，明确视觉分析的主次重心：

* **编程/开发场景 (例如 VS Code, PyCharm)**：
    * **主要关注点**：终端报错日志、代码红线警告、异常堆栈信息。
    * **次要关注点**：正常运行的代码逻辑、文件目录树。

* **硬核动作与魂类游戏场景 (例如《艾尔登法环》、《怪物猎人》、《鬼泣》、《忍龙》)**：
    * **主要关注点**：玩家当前血量/精力条（绿条）、Boss 生命值与阶段状态、血瓶/恢复道具剩余数量、武器锋利度/耐久度、华丽度评分（如 DMC 的 Style 评价）。
    * **次要关注点**：背景环境细节、普通小怪的分布、游戏风景。

* **开放世界与剧情 RPG 场景 (例如《巫师3》、《赛博朋克2077》、《对马岛之魂》)**：
    * **主要关注点**：当前剧情对话框与字幕（用于 AI 陪伴吐槽剧情）、任务目标提示、潜行警戒度指示器（是否被敌人发现）、小地图/雷达导航路径。
    * **次要关注点**：天气系统（雨雪昼夜）、环境路人 NPC、背包整理界面。

* **MOBA 与竞技对战场景 (例如《英雄联盟》、《永劫无间》)**：
    * **主要关注点**：当前战绩面板 (KDA/补刀/存活状态)、终极技能（大招）冷却情况、小地图视野（是否有敌人消失或包抄）、团队血量健康度、护甲/护盾碎裂提示。
    * **次要关注点**：网络延迟 (Ping)、游戏帧率 (FPS)、公屏聊天记录（防剧透或防干扰）。

* **策略塔防与动作刷宝场景 (例如《明日方舟》、《命运2》、《鸣潮》)**：
    * **主要关注点**：部署费用/能量条（Cost/超能力）、角色连携/QTE触发提示、技能 CD 倒计时、弹药余量、高难副本的机制倒计时或 Boss 狂暴提示。
    * **次要关注点**：抽卡结算画面（除非需要 AI 恭喜或安慰）、关卡掉落结算面板。

## 3. 核心机制：动态提示词与增量标记
基于上述场景规则，本模块在打包数据时会执行以下强化操作：
1.  **动态提示词注入 (Dynamic Prompt Injection)**：将场景规则翻译为对大模型的直接指令。例如，当检测到 IDE 进程时，自动在数据包中追加指令：`"请重点提取画面中的报错信息，忽略正常的代码块。"`
2.  **状态变化标记 (Status Delta - 预留)**：预留增量字段 `status_delta`，用于标记关键事件的发生（如“画面判定：等级从5升至6”），让后续的状态机能够对“变化”做出反应，而非仅仅描述“现状”。

## 4. 上下文清洗与隐私保护 (Context Cleansing & Privacy)
在封装数据前需进行轻量级过滤：
* **敏感信息脱敏**：如果检测到 `process_name` 属于高敏软件（如密码管理器 `1Password`、支付宝/银行插件），本模块将强制拦截本次图像载荷，将 `image_data` 替换为空，仅传递“用户正在处理敏感事务”的文本状态，确保隐私绝对安全。
