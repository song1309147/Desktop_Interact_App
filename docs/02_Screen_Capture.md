# 桌面互动软件 - 02_实时屏幕读取并交互文档

## 1. 模块概述
本模块负责系统“视觉感知”的原始数据采集。它犹如挂件的“眼睛”，在后台以极低的系统资源占用率静默运行。通过“事件驱动（切屏）”与“时间驱动（定时）”双引擎机制，智能抓取全屏画面，并在内存中完成预处理后交由视觉大模型（VLM）进行分析。

## 2. 核心触发引擎设计
为避免频繁截图导致 API Token 耗尽或打断用户体验，本模块采用双引擎并行监听，并引入防抖（Debounce）机制与严格的边界控制。

### 2.1 引擎 A：事件驱动（窗口切换监听）
* **触发条件**：后台每秒轮询当前操作系统的“前台激活窗口句柄/标题”。一旦发现窗口发生变更（例如从 `VS Code` 切换至其它软件），即触发截图请求。
* **防抖限制（冷却期）**：**强制 15 秒冷却**（遵循全局最小间隔限制）。如果距上一次任何形式的截图触发不足 15 秒，本次切屏事件将被丢弃。这有效防止了快速按 `Alt+Tab` 查找资料时引发的疯狂截图。

### 2.2 引擎 B：时间驱动（动态定时兜底）
* **触发条件**：当长时间停留在同一个窗口（沉浸式工作或游戏）时，引擎 A 不会被触发。此时由引擎 B 的倒计时器接管。倒计时归零时，强制触发一次全屏截图。
* **动态配置与边界控制**：倒计时的时长（最大静默抓取间隔）不写死在代码中，而是每次启动或截图后，动态调用 `utils/config_loader.py` 读取本地 `settings.json` 文件的配置。
  * **默认最大值（兜底策略）**：如果前端未传递参数或未进行自定义设置，系统**默认最大静默抓取间隔为 5 分钟**。
  * **默认最小值（频率保护）**：系统设定全局下限，**默认最小值为 15 秒**。无论是前端自定义输入，还是系统内部的自动设置调整，**抓取间隔均严格限制不得低于 15 秒**，以防止系统资源耗尽或 API 滥用。

## 3. 性能优化与图像处理流 (Performance & Processing)
为确保不对运行帧率或电脑性能造成任何影响，截图生命周期严格遵循以下规范：

1. **极速抓取**：采用底层高效截图库（如 `mss` 或 `pyautogui`），仅获取当前屏幕原始分辨率画面的像素阵列，过程需控制在 100ms 以内。
2. **内存流转 (BytesIO)**：**禁止将临时截图写入硬盘！** 抓取到的图像直接保存在运行内存的缓冲区中，实现“阅后即焚”，彻底消除磁盘 I/O 碎片损耗。
3. **体积压缩 (Resizing & Compression)**：
   * 大模型对高清原图收费极其昂贵且传输极慢。
   * 在内存中将全屏图像进行等比例缩放（限制最长边不超过 1080px 或 720px，足以让模型看清 UI 与文字）。
   * 转换为高压缩比的 `JPEG` 格式，单张图片体积严格控制在 **300KB 以内**，随后再发送给网络模块。

## 4. 模块数据交互与依赖
本模块不直接呼叫 API 或操作界面，而是作为数据提供者：
* **向外输入**：依赖 `utils/config_loader.py` 获取截图频率偏好与配置边界。
* **向外输出**：当满足截图条件（且经过 15 秒下限校验）及预处理完成后，将 `[压缩后的图像内存对象]` 和 `[当前窗口名称]` 作为变量，传递给 **状态机 (State Machine)** 模块，由状态机决定是否呼叫大模型。
