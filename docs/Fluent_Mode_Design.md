# 桌面互动软件 - 功能设计文档

## 模块名称：2.2 流畅模式（Fluent Mode）

### 1. 模块概述
本模块旨在为用户提供类似真实口头交流的连续、流畅对话体验。系统将完全接管收音与播放的流程控制，大幅减少用户的手动操作（如点击录音、点击发送）。适用于需要高频互动、练习口语流利度或沉浸式陪伴的场景。

**核心设计原则：**
* **语音驱动：** 弱化键鼠输入，以语音为核心交互媒介。
* **状态自动化：** 自动侦测用户的发声起始与结束，自动流转对话状态。
* **兜底机制：** 具备超时判定与主动引导能力，避免冷场。

### 2. 核心交互状态流转
在流畅模式下，软件在任何时刻都处于以下四个核心状态之一：

| 状态名称 | 触发条件 | 系统行为 | 视觉反馈建议（UI层） |
| :--- | :--- | :--- | :--- |
| **S1: 播放中 (Playing)** | 接收到大模型回复并开始 TTS | 锁定麦克风，播放音频流。 | 角色说话动画 / 音波律动特效 |
| **S2: 监听中 (Listening)** | TTS 播放结束，或用户主动触发 | 开启麦克风，启动 VAD（语音端点检测），**同时启动 10 秒倒计时器**。 | 麦克风高亮 / 角色呈倾听姿态 |
| **S3: 识别与推理 (Processing)** | VAD 判定用户断句，或倒计时结束且有声音 | 关闭麦克风，调用 STT -> RAG -> Qwen API。 | 角色呈思考状态 / 加载动画 |
| **S4: 超时引导 (Timeout)** | 10秒倒计时结束且 VAD 未检测到有效语音 | 强制中断监听，调用预设提示语库（或极简 Prompt 呼叫大模型）生成引导句，进入 S1 播放。 | 角色呈现疑惑或询问表情 |

### 4. 关键技术难点与准备方向
1. **VAD (Voice Activity Detection) 模块：** 不能仅靠音量判断，需排除环境噪音。建议后续接入轻量级开源模型（如 Silero VAD）进行精准断句。
2. **异步与多线程 (Threading/Asyncio)：** 录音、计时器、API 请求必须在后台线程运行，避免阻塞主 UI 界面导致程序“卡死”。
3. **音频流状态监控：** 需监听音频播放库的“播放完毕”事件，以此作为自动激活麦克风的触发器。
