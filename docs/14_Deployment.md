# 桌面互动软件 - 14_软件打包与发布文档 (Deployment & Distribution)

## 1. 模块概述
本模块定义了如何将零散的 Python 脚本、Web 前端资源（HTML/CSS/JS）、配置文件以及本地 AI 模型，固化并封装为一个普通用户可直接双击运行的桌面应用程序（如 Windows 的 `.exe`）。本阶段的核心挑战是处理静态资源的路径映射以及本地大模型权重的打包。

## 2. 核心打包工具选型
统一采用 **`PyInstaller`** 作为核心封装工具。
* **策略选择：单文件夹模式 (`-D` 或 `--onedir`)**。
* **严禁使用单文件模式 (`-F` 或 `--onefile`)**：由于本项目包含本地语音识别模型及大量的 Web 渲染组件，强行打包成单一 `.exe` 会导致每次启动时在后台解压数 GB 的临时文件，使得软件启动极其缓慢（可能需要几分钟），且极易触发杀毒软件的内存拦截。因此，必须打包为一个包含 `.exe` 及周边依赖库的“绿色免安装文件夹”。

## 3. 资源路径映射（核心排错指南）
在交给 AI 编写代码时，必须强制要求 AI 封装一个全局的“资源路径获取函数”。
* **问题背景**：打包后，软件运行时的绝对路径会发生变化（被定向到 `sys._MEIPASS` 临时目录），导致原本代码中的相对路径（如 `./assets/ui.html`）全部失效报错。
* **解决方案**：所有读取前端 HTML 文件、Q 版角色图片、设置 `config.json` 甚至读取 SQLite 数据库的操作，都必须经过如下逻辑判断：
  ```python
  import sys
  import os

  def get_resource_path(relative_path):
      """打包兼容的路径寻址函数"""
      if getattr(sys, 'frozen', False):
          # 如果是打包后的运行环境
          base_path = sys._MEIPASS
      else:
          # 如果是开发环境
          base_path = os.path.abspath(".")
      return os.path.join(base_path, relative_path)
  ```

## 4. 配置文件生成与大模型剥离
为保证朋友拿到软件后可以自由设置，必须处理好以下两类特殊文件：
* **可见配置（不可打包进内部）**：像 `.env`（存 API Key 的地方）或 `config.json`（存截屏频率和性格的地方），绝不能用 PyInstaller 打包进代码里。必须在代码中设定：如果软件启动时发现外面没有这些文件，就自动在 `.exe` 同级目录下生成一份默认的空配置文件，供用户填写。
* **庞大的模型权重 (Local Models)**：如果使用了本地的 FunASR 模型，务必在 PyInstaller 的 `.spec` 文件中，通过 `datas` 参数将模型权重的文件夹完整映射进打包目录中，否则语音转写功能将彻底失效。

## 5. 分发与运行须知
* **压缩分发**：将打包生成的整个 `dist/项目名` 文件夹使用 ZIP 或 7z 压缩后发给用户。
* **环境提示**：尽管打包后不需要用户安装 Python，但某些底层库（如 `webrtcvad` 或截屏底层）可能依赖系统自带的 C++ 运行库。可在压缩包内附带一份微软官方的 `VC++ Redistributable` 安装包作为防崩溃的兜底保障。
* **信任白名单**：由于缺乏昂贵的企业级代码签名证书，向用户说明在解压和运行时如果遇到 Windows Defender 的拦截（如报 Trojan:Win32/Wacatac 等泛型启发式威胁），需手动将其加入信任区（白名单）。
