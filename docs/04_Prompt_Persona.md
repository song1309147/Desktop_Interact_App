# 桌面互动软件 - 04_提示词工程与角色设定文档

## 1. 模块概述
本模块负责定义挂件的“灵魂”。它通过**模块化提示词拼装（Modular Prompt Assembly）**技术，将用户的个性化设定、实时的视觉上下文以及长期的交互记忆缝合在一起。本模块确保挂件不仅能看懂屏幕，还能以符合设定且逐渐稳定的语气与用户交流。

## 2. 角色设定与初始化 (Persona Initialization)
应用支持首次启动设定及前端随时更改。系统维护一套“角色画像（Persona Profile）”：

* **默认设定 (Default Profile)**：
    * **基础属性**：女性、温柔、青年。
    * **性格倾向**：包容、鼓励式、理性但不失温度。
    * **自称与语气**：通常使用“我”或用户设定的特定称呼，语尾柔和，不使用攻击性词汇。
* **前端自定义映射 (Frontend Mapping)**：
    * **身份定义**：如“妹妹/哥哥/助手/战友”等（决定称呼语系）。
    * **性格标签**：可选择“傲娇、高冷、热血、幽默”等标签，每个标签对应一组 System Prompt 关键词。
    * **首次启动逻辑**：用户首次打开软件将强制弹出“性格捏脸”界面；若用户选择“跳过”，则自动加载上述默认设定。

## 3. 提示词拼装架构 (Prompt Architecture)
发送给视觉大模型（VLM）的最终 Prompt 由四个动态积木块拼接而成，确保逻辑清晰且易于维护：

| 积木块名称 | 来源 | 说明 |
| :--- | :--- | :--- |
| **A. 核心人设 (Core)** | 前端设置/默认设置 | 静态部分。定义基本身份、性别、禁忌事项。 |
| **B. 演化记忆 (Evol.)** | 性格收敛引擎 | 动态部分。包含根据历史对话总结出的用户偏好和语气习惯。 |
| **C. 视觉上下文 (Vision)** | 03号模块输出 | 实时部分。注入当前软件场景（如编程/游戏）及主次关注点指令。 |
| **D. 实时任务 (Task)** | 系统触发逻辑 | 指令部分。如：“请对目前的报错给出安慰”或“简短评价当前的战况”。 |

## 4. 性格收敛引擎：50 轮总结机制 (Personality Convergence)
为实现“性格随交流趋向于稳定”的核心需求，系统内置自动总结反馈逻辑：

1.  **计数触发**：系统本地维护一个对话计数器，每积累 **50 条** 对话记录（用户输入 + 挂件回复）后触发一次总结任务。
2.  **镜像分析**：调用大模型对这 50 条历史记录进行异步分析，提取以下关键特征：
    * **用户偏好提取**：用户对哪类互动反应积极？喜欢被夸奖、被吐槽还是被提醒？
    * **语气稳定性修正**：在过去交互中，挂件表现最自然的语气词、自称和情绪波动范围是什么？
3.  **画像持久化**：将分析结果汇总并更新至 `persona_evolution.json`。这个“进化后的画像”将作为**积木块 B** 长期存在，使挂件的性格在 50 条、100 条、150 条对话后逐渐“收敛”，最终形成与用户高度契合的专属性格。

## 5. 视觉分析指令集成 (Vision Instruction Integration)
本模块根据 03 模块提供的 `primary_focus`（关键信息）动态调整提示词优先级：

* **编程场景下**：若 `primary_focus` 包含“报错”，Prompt 自动切换为“技术支持与情绪安抚”模式，引导大模型分析错误原因。
* **游戏场景下**：若 `primary_focus` 包含“战绩/剧情”，Prompt 切换为“同伴视角”，引导大模型进行共情反馈或战况点评。
