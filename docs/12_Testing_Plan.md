# 桌面互动软件 - 12_测试文档 (Testing Plan & QA)

## 1. 模块概述
本模块定义了软件在开发后期必须经历的“质量保证（QA）与压力测试”标准。区别于常规软件，本项目的测试核心目标是**“极低侵入性验证”与“极端场景下的鲁棒性（防崩溃能力）”**。测试用例将围绕高负载游戏环境、高频状态切换、复杂的音频环境以及网络异常四个维度展开。

## 2. 核心测试维度与用例 (Test Cases)

### 2.1 性能与无感化测试 (Performance & Stealth Test)
**目标**：验证 02 模块（截屏）与 11 模块（底层架构）是否会影响宿主机的游戏体验。
* **用例 A（高负载截屏）**：运行 3A 大作（如《艾尔登法环》、《赛博朋克 2077》）并保证 GPU 满载。在此期间触发软件的连续高频截图（如每 15 秒一次）。
  * **通过标准**：游戏内帧率（FPS）波动不超过 3 帧，无明显的微卡顿（Micro-stuttering）。
* **用例 B（UI 防误触穿透）**：在《英雄联盟》或《永劫无间》等需要高频点击的游戏中，将鼠标反复滑过并点击挂件所在的屏幕区域。
  * **通过标准**：点击必须 100% 穿透挂件，精准作用于游戏画面，绝不触发挂件的拖拽或实体化。

### 2.2 状态机混沌测试 (State Machine Chaos Test)
**目标**：验证 08 模块的“交通规则”是否严密，打断机制是否生效。
* **用例 A（快速切屏免疫）**：在游戏与桌面之间，以 1 秒 1 次的频率连续 `Alt+Tab` 切换 10 次。
  * **通过标准**：3 秒免疫规则生效，在此期间软件不得发起任何无效的截图和 API 请求。
* **用例 B（物理打断强制重置）**：当挂件正在滔滔不绝播放长语音时，将鼠标移至挂件上方并快速晃动。
  * **通过标准**：音频瞬间掐断，气泡立即消失，系统状态无缝强制重置回 `IDLE`（休眠），且不影响下一次正常唤醒。

### 2.3 听觉与 VAD 降噪测试 (Audio Environment Test)
**目标**：验证 05 模块能否在嘈杂环境中精准剥离人声。
* **用例 A（机械键盘免疫）**：在进入 10 秒语音交互窗口期时，不说话，但高强度敲击机械键盘与鼠标（模拟打字或激烈操作）。
  * **通过标准**：VAD 模块不应被激活，10 秒结束后麦克风必须准时自动关闭并休眠。
* **用例 B（混杂人声提取）**：在游戏背景音量处于中等水平时，以正常音量对麦克风说话。
  * **通过标准**：本地 STT 能准确识别指令内容，且未将游戏背景音中的角色台词误识别为用户指令。

### 2.4 网络异常与容错测试 (Network Fallback Test)
**目标**：验证 07 模块的“断联兜底机制”，确保软件永不闪退。
* **用例 A（API 超时/拔网线）**：在软件刚截完图、进入 `OBSERVING` 状态准备发送请求的瞬间，人为断开电脑网络。
  * **通过标准**：软件等待 10~15 秒后，优雅退出等待状态，气泡提示“网络似乎断开了”或无声回落至休眠，**绝对禁止出现代码级 Crash（崩溃闪退）**。
* **用例 B（密钥失效模拟）**：在设置面板中输入错误的 API Key 并保存，随后触发截图交互。
  * **通过标准**：底层拦截 `401 Unauthorized` 错误，挂件通过语音或文字友好提示用户检查密钥，程序继续保持后台存活。
