# 桌面互动软件 - 06_交互表现文档

## 1. 模块概述
本模块是系统与用户产生情感连接的“物理外壳”。它负责将内部复杂的 AI 逻辑（文本、情绪、状态）转化为直观、生动且**绝对不干扰正常电脑使用**的视觉表现。秉承“跨界陪伴”的设计初衷，本模块旨在打造一个能适应从 3A 动作大作到复杂策略游戏等无数种桌面环境的“赛博好友”，让每一次互动都自然、灵动且克制。

## 2. 核心渲染机制：无感化与防误触
为了确保在激烈操作（如动作类、FPS竞技类游戏）或专注办公时，挂件不会成为累赘，底层窗口渲染严格遵循以下规范：

### 2.1 全透明与事件穿透 (Mouse Passthrough)
* **渲染层级**：采用系统最高层级置顶（Top-Most）无边框透明窗口渲染，确保挂件始终可见。
* **物理防误触**：在默认的“静默观察”或“播放发声”状态下，开启**全局鼠标穿透**（Click-Through）。用户的任何点击、拖拽都会直接穿透挂件，作用于后方的游戏或工作软件。
* **交互接管**：仅当用户按下特定的修饰键（如 `Alt` + 鼠标悬停），或主动触发设定界面时，挂件才恢复实体碰撞体积，接收用户的点击反馈。

### 2.2 半透明退让状态 (Opacity Yielding)
* **视觉减负**：当系统处于长时间无交互的闲置状态时，挂件及 UI 元素的透明度自动降低至 40%-50%，半隐入背景，减少视觉疲劳。当 05 模块传来 `STATE_SPEAKING` 或 `STATE_LISTENING` 信号时，瞬间恢复 100% 不透明度。

## 3. 智能 UI 避让与动态气泡 (Smart Dodging & Dynamic UI)
鉴于用户运行的软件/游戏 UI 布局千变万化，交互气泡框需具备“自我意识”：

### 3.1 动态锚点与反向弹出
* 系统提供多个可记忆的“停靠锚点”（如屏幕四角或边缘）。
* **智能避让边缘**：文字气泡会根据挂件所在屏幕的位置自动计算弹出方向。例如挂件在右下角，气泡自动向左上角延展，绝不超出屏幕边界。
* **场景感知避让**：结合 03 模块的视觉上下文，气泡应尽量避开当前软件的常规高频操作区（例如 MOBA 游戏的右下角小地图，或动作 RPG 游戏的左上角血条区）。

### 3.2 动态停留时长 (Dynamic Linger Time)
告别死板的固定秒数消失机制。气泡展示时长根据大模型返回的文本长度动态计算：
* `展示时长 = 基础动画时间 (1.5s) + [字数 × 单字阅读耗时 (约 0.25s)]`
* 确保用户刚好舒适地看完吐槽或提示，随后气泡以淡出（Fade-out）动画优雅消失。

## 4. 多模态表现同步 (Multimodal Synchronization)
要营造“真实陪伴感”，视觉与听觉必须高度咬合：

### 4.1 唇音同步 (Lip-Sync)
* 彻底摒弃简单的“张嘴循环动画”。Live2D 模型的口型开合幅度（Mouth Open）必须与 05 模块 TTS 输出的实时音频振幅（Volume/RMS）直接绑定。声音大则嘴张大，声音停则嘴闭合。

### 4.2 情绪动画映射 (Emotion Mapping)
* 解析 05 模块传来的 `emotion_tag`。
* 将大模型定义的情绪（如：激动、调侃、安慰、思考）映射至 Live2D 的预设动作组（`.motion3.json`）和表情组（`.exp3.json`）。例如：当 AI 判定用户打出极限操作并进行夸奖时，伴随语音，挂件同时触发“鼓掌”或“星星眼”动作。

## 5. 状态机视觉反馈 (Visual State Indicators)
精确反映系统当前在做什么，避免用户产生“它是不是死机了”的错觉：
1. **监听中 (Listening)**：10 秒交互窗口期内，挂件做出微小的“侧耳”动作，气泡处显示微弱的脉冲呼吸灯，示意“正在听你说话”。
2. **思考中 (Processing)**：捕获语音或截图后等待 AI 返回期间，显示旋转的“加载”小图标或托腮思考动作。
3. **表达中 (Speaking)**：气泡弹出，语音播放，唇音同步激活。
