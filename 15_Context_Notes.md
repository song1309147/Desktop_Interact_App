# 桌面互动软件 - 15_AI 辅助开发准则与全局备注 (AI Coding Guidelines & Roadmap)

## 1. 模块概述
本文档是本项目所有开发工作的“最高宪法”与“需求停车场”。作为由人类（产品/架构师）与 AI（主力程序员）协同开发的软件，本文档规定了 AI 生成代码时的绝对纪律，并收录了不属于 V1.0 版本的未来高阶需求。**在进行任何模块的代码生成前，AI 必须优先读取并严格遵守本指南。**

## 2. 全局编码纪律 (Mandatory Coding Conventions for AI)
AI 在为本软件输出任何代码时，必须无条件遵守以下规则：
* **模块化解耦**：严格遵循 01-14 文档设定的边界。例如，前端界面（09 模块）代码中绝对不允许混入截屏（02 模块）的底层逻辑，必须通过明确的接口或事件总线调用。
* **强制中文注释**：所有核心类、状态机切换逻辑、复杂的正则表达式（特别是 03 模块的隐私清洗部分），**必须**在上方添加通俗易懂的中文注释，确保非计算机专业背景的架构师能够完全读懂其运行机制。
* **异步与防阻塞优先**：软件的灵魂是“不干扰日常打游戏”。任何涉及网络请求（如大模型 API 调用）、磁盘读写（如 SQLite 存储）、截图与图像处理的操作，**强制使用 `asyncio` 异步语法或独立线程**，绝对禁止阻塞 UI 主线程导致画面卡顿。
* **防御性编程 (优雅降级)**：拦截一切可能导致软件闪退的异常（如断网、API 欠费、截图权限丢失）。捕获异常后，需将其转化为控制台日志或触发 08 模块的休眠状态，软件本身必须保持存活。

## 3. 人机协作标准暗号 (Meta-Prompts Templates)
为降低沟通成本，人类架构师在指派任务时将使用以下标准化句式，AI 需按规定格式响应：

* **[指令 A] 模块初始化**：“请读取 `XX_Module.md`，严格按照文档要求，仅输出该模块的 Python/HTML 核心代码。不要做任何多余的解释，代码需可直接复制运行。”
* **[指令 B] 报错抢修**：“运行出错，报错信息如下：`[粘贴报错内容]`。请直接给出需要在终端执行的修复指令（如 pip install），或直接给出修改后的代码片段，无需长篇大论分析原理。”
* **[指令 C] 语境重置**：“忘记之前的错误思路，重新回顾 15_Context_Notes 及当前模块文档，以‘听从哥哥指令的首席程序员’身份，重新编写这部分逻辑。”

## 4. V2.0 未来愿景池 (Future Roadmap - 需求停车场)
以下功能属于“超酷但不属于第一阶段”的设想。在 V1.0 跑通之前，严禁将以下需求混入当前代码库，仅作为备忘：
1. **真实骨骼动画接入**：未来探讨抛弃 Q 版静态形变，接入真正的 Live2D SDK (`.moc3` 格式)，实现跟随鼠标视线移动和更细腻的情感表达。
2. **深度游戏 API 挂钩**：针对《英雄联盟》或《怪物圣所》等常玩游戏，未来尝试接入官方或第三方内存读取 API（如 LCU API），实现毫秒级的击杀播报与状态监控，逐步降低对视觉截图的依赖。
3. **无限记忆向量库 (RAG)**：当 SQLite 中的 1000 条对话不够用时，未来引入 ChromaDB 等本地向量数据库，让赛博伴侣能真正记住几个月甚至几年前两人聊过的游戏梗和细节。
4. **桌面级物理交互**：允许 Q 版角色“坐”在用户打开的 VS Code 窗口边缘，或者在用户拖动窗口时产生物理碰撞效果。
