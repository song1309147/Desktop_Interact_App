# 桌面互动软件 - 项目总文档 (Master PRD)

## 1. 项目愿景与产品定位 (Vision & Positioning)
* **产品名称**：桌面互动软件（暂定名，后续根据角色设定更改）
* **产品形态**：悬浮于操作系统的无边框桌面挂件（如 Q 版人物立绘/动画）。
* **核心卖点**：从“被动唤醒”转向“主动陪伴”。软件能够定时读取用户的屏幕画面，结合当前上下文（如游戏战局、编程代码）主动做出符合设定的情绪化评价，并有条件地开启语音交互，打造沉浸式的赛博伙伴体验。

## 2. 核心交互流 (Core Interaction Loop)
本软件的生命周期围绕以下核心状态机流转：
1. **定时/事件触发 (Trigger)**：程序静默运行，根据设定的时间间隔（如每 30 秒）或特定屏幕事件，截取当前屏幕画面。
2. **视觉感知与主动评价 (Analyze & Comment)**：
   * 将截图与系统提示词发送至视觉大模型（VLM）。
   * 模型生成符合角色设定的评价文本（如吐槽、夸奖、提示）。
   * 触发文本转语音（TTS）进行播报，同时前端 UI 展示对应的说话动画。
3. **条件监听 (Conditional Listen)**：
   * 播报结束瞬间，软件自动开启麦克风，进入限时监听状态（如 5-8 秒）。
4. **分支响应 (Branching)**：
   * **分支 A（用户回复）**：若在监听期内检测到用户语音输入，则启动语音转文字（STT），将文本送入大语言模型（LLM）生成回复，并再次通过 TTS 播报。完成此轮深度交互后退出监听。
   * **分支 B（用户沉默）**：若限时内未检测到有效语音，麦克风平滑关闭，UI 恢复待机动画，进入下一轮定时触发周期。

## 3. 功能模块全景图 (Feature Overview)
* **前端展示模块**：无边框透明窗口绘制、拖拽逻辑、状态动画切换。
* **屏幕捕获模块**：低性能消耗的全屏或区域截图机制。
* **智能中枢模块**：对接多模态大模型（VLM/LLM），处理图像理解、上下文记忆与回复生成。
* **语音处理模块**：集成人声端点检测（VAD）、语音转文字（STT）与文字转语音（TTS）。
* **状态调度模块**：全局状态机，确保截图、播报、监听三大行为互斥且有序流转。

## 4. 核心受众与使用场景 (Target Audience & Use Cases)
* **目标用户**：希望在独自游戏、办公或学习编程时获得陪伴感的用户。
* **典型场景**：
  * **游戏场景**：用户打出高光操作或阵亡时，挂件结合屏幕战绩给出夸奖或嘲讽。
  * **编程场景**：用户代码频繁报错卡住时，挂件读取屏幕报错信息并给出语音提示。
